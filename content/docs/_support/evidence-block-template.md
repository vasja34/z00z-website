## Evidence Block Template

Use this private snippet only as an authoring aid for public docs pages that
need a consistent evidence footer.

### Required Inputs

- Primary whitepaper source files and exact section anchors
- Any live repository evidence that supports a narrower implementation claim
- A one-line maturity statement: `live repo evidence`, `target architecture`,
  or `open research`

### Suggested Footer Shape

1. Start with the user-facing claim the page just made.
2. Attach the exact whitepaper file and section that justifies the claim.
3. Add repo-local evidence only when the repository actually proves the point.
4. End with a `Read Next` suggestion that keeps the learning path honest.

### Non-Goals

- Do not turn this private snippet into a public route.
- Do not store marketing copy or unsourced promises here.
- Do not imply shipped implementation when the corpus only defines a target.
